# Agent Handoff Guide

本文档用于不同角色 agent 之间交接工作。`docs/agents.md` 定义“谁负责什么”，本文定义“如何交接、如何越界申请”。

## 当前默认角色

当前对话中的默认角色为：主策划。

主策划负责维护玩法目标、系统边界、设计取舍、验收口径和文档结构。主策划不默认浏览 UI 图片资源、导出素材、代码实现、临时文件或音频资源，除非任务明确要求或获得许可。

## 交接原则

- 每个角色只回溯自己职责范围内的文件。
- 需要越界读取或修改时，先获得项目负责人许可。
- 跨角色交接优先写清楚“已确认口径”和“待确认问题”。
- 历史文件只作参考，不得覆盖最新明确口径。

```md
我需要越界读取/修改以下内容：

- 角色：

```

获得明确许可后才能继续。

- 当前主线是首批 10 英雄技能、3x3 站位、英雄小队、英雄主动技能自动释放和军令卡系统 V1。
- 旧版装备、旧版战备 3 选 1、旧版羁绊和旧版 24 人英雄池不作为当前依据。
- 任何角色接手英雄技能相关工作时，优先读取 `first-batch-10-heroes-skill-config-v1.md`、`tactical-command-card-system-v1.md` 和 `hero-skill-template-guideline.md`；旧 8 英雄文档只用于回溯已通过的玩家阅读文本。

## Handoff: 通用军令战斗 HUD 与卡牌设计（已暂停）

> 状态：本任务基于固定三卡 V0，已被军令卡系统 V1 替代。保留任务原文供回溯，不得继续执行或据此产出 UI。

- 发起角色：主策划
- 接收角色：UI/UX
- 日期：2026-07-13
- 当前目标：为通用军令 V0 输出战斗 HUD 布局、三张卡牌视觉规范与完整交互状态说明，供后续 P0 原型拼接使用。
- 已确认口径：英雄已解锁主动技能始终自动释放；玩家的手动参与只来自固定三张通用军令，不存在英雄技能卡池、抽牌、刷新、弃牌、费用或倒计时自动消失。
- 已确认卡牌：护阵令、压阵令、破势令。普通波与精英波每波最多使用 1 张；Boss 波每波最多使用 2 张，且必须使用两张不同的卡；同一卡每波最多使用 1 次。
- 已确认交互：需要目标的军令只有在存在合法目标时可用；无合法目标时显示禁用且不消耗本波次数。卡牌不会自动释放或自动消失；已使用卡本波禁用，下波恢复可用。
- 需要接收角色处理：
  - 设计战斗 HUD 中三张固定军令卡的布局与信息层级。
  - 为三张卡定义卡面视觉规范，并明确可用、禁用、已使用、目标选择四种状态。
  - 输出普通/精英波状态示例，以及 Boss 波使用 0、1、2 张后的状态示例。
  - 明确目标选择态的视觉反馈，包括合法目标、非法目标、取消选择和释放确认。
- 交付物：`docs/design/ui-ux/` 下的军令 HUD/交互设计文档；如需视觉稿，按现有 UI 资源工作流产出预览并更新 UI 索引或资源 manifest。
- 越界读取/修改需求：获项目负责人授权，可读取 `docs/design/tactical-command-v0.md`、`docs/design/general-rules.md` 与本任务相关的 UI 文档；不得修改核心玩法、军令效果、使用次数或英雄技能规则。如发现 UI 无法承载的规则问题，只记录问题并回交主策划确认。

## Handoff: 军令卡系统 V1.1 的 UI 准备（可发起）

- 发起角色：主策划
- 接收角色：UI/UX
- 日期：2026-07-15
- 当前目标：为军令市场、主动军令 HUD、军略图鉴、Set 进度与战前路目标选择准备 UI/UX 方案；不修改核心玩法和卡牌数值。
- 已确认口径：备战阶段为 3 报价市场，同次备战可购买多张并刷新。玩家从已拥有主动军令中选择至多 3 张进入 HUD；普通/精英波 2 点军令点，Boss 波 3 点。被动军略进入后台图鉴；Set 自动吞噬/自动进阶，高阶结果不进市场。
- 已确认表现要求：市场卡显示 Set 名称、进度、缺失组件、融合结果和“新解锁”来源；选路军令按战前上/中/下路归属显示，自动锁定目标须高亮预览。飞骑、伏兵使用临时战术表现，不调度英雄队伍移动。
- 未完成事项：基础军资经济、刷新价格、报价权重、购买后补位、视觉稿和完整交互状态。
- 接收角色优先读取：`docs/design/tactical-command-card-system-v1.md`、`docs/design/tactical-command-card-catalog-v1.md`、`docs/design/general-rules.md`。
- 图标资源任务：先读取 `docs/design/ui-ux/command-card-v1-handoff.md` 的“军令图标资源包 V1”；按效果图审美确认、无文字清稿、工程切片顺序推进，不得直接批量生成最终切片。
- 越界读取/修改需求：UI/UX 需要修改 UI 文档或资源时，先按本文档的越界申请模板获得项目负责人许可；不得修改核心规则、卡牌效果、军令点和数值。
