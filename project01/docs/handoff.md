# Agent Handoff Guide

本文档用于不同角色 agent 之间交接工作。`docs/agents.md` 定义“谁负责什么”，本文定义“如何交接、如何越界申请”。

## 当前默认角色

当前对话中的默认角色为：主策划。

主策划负责维护玩法目标、系统边界、设计取舍、验收口径和文档结构。主策划不默认浏览 UI 图片资源、导出素材、代码实现、临时文件或音频资源，除非任务明确要求或获得许可。

## 交接原则

- 每个角色只回溯自己职责范围内的文件。
- 需要越界读取或修改时，先获得项目负责人许可。
- 跨角色交接优先写清楚“已确认口径”和“待确认问题”。
- 历史文件只作参考，不得覆盖最新明确口径。

获得明确许可后才能继续。

## Handoff: 通用军令战斗 HUD 与卡牌设计（已暂停）

> 状态：本任务基于固定三卡 V0，已被军令卡系统 V1 替代。保留任务原文供回溯，不得继续执行或据此产出 UI。

- 发起角色：主策划
- 接收角色：UI/UX
- 日期：2026-07-13
- 当前目标：为通用军令 V0 输出战斗 HUD 布局、三张卡牌视觉规范与完整交互状态说明，供后续 P0 原型拼接使用。
- 已确认口径：英雄已解锁主动技能始终自动释放；玩家的手动参与只来自固定三张通用军令，不存在英雄技能卡池、抽牌、刷新、弃牌、费用或倒计时自动消失。
- 已确认卡牌：护阵令、压阵令、破势令。普通波与精英波每波最多使用 1 张；Boss 波每波最多使用 2 张，且必须使用两张不同的卡；同一卡每波最多使用 1 次。
- 已确认交互：需要目标的军令只有在存在合法目标时可用；无合法目标时显示禁用且不消耗本波次数。卡牌不会自动释放或自动消失；已使用卡本波禁用，下波恢复可用。
- 需要接收角色处理：
  - 设计战斗 HUD 中三张固定军令卡的布局与信息层级。
  - 为三张卡定义卡面视觉规范，并明确可用、禁用、已使用、目标选择四种状态。
  - 输出普通/精英波状态示例，以及 Boss 波使用 0、1、2 张后的状态示例。
  - 明确目标选择态的视觉反馈，包括合法目标、非法目标、取消选择和释放确认。
- 交付物：`docs/design/ui-ux/` 下的军令 HUD/交互设计文档；如需视觉稿，按现有 UI 资源工作流产出预览并更新 UI 索引或资源 manifest。
- 越界读取/修改需求：获项目负责人授权，可读取 `docs/design/tactical-command-v0.md`、`docs/design/general-rules.md` 与本任务相关的 UI 文档；不得修改核心玩法、军令效果、使用次数或英雄技能规则。如发现 UI 无法承载的规则问题，只记录问题并回交主策划确认。
