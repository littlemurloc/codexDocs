# 阵容编成 / 羁绊页签 v01

## 资源

- 预览图：`assets/ui/preview/lineup-synergy-v01.png`
- 状态：已确认
- 界面：阵容编成 / 羁绊页签
- 入口：阵容编成页顶部页签中的“羁绊”，或底部“羁绊效果预览”摘要
- 设计方向：继承 `lineup-formation-v01` 的同层级切页结构

## 界面定位

本页是阵容编成内的羁绊视角，不是独立养成页、弹窗、抽屉或武将配置页。玩家仍处在“阵容编成”流程中，只是从“出战”切换到“羁绊”页签，用于查看当前上阵武将触发了哪些阵营与职业羁绊。

羁绊是多复合关系，阵营羁绊和职业羁绊分别计算，多个羁绊可以同时生效。本页需要避免把羁绊表现成单选天赋、唯一主羁绊或付费解锁列表。

## 已确认设计规则

### 继承阵容编成页层级

页面标题仍为“阵容编成”，顶部页签保留：

```text
出战 / 羁绊 / 预设
```

当前高亮“羁绊”。右侧武将列表、底部“保存阵容”主操作和资源栏均继承 `lineup-formation-v01`，让玩家明确这是同一界面的切页内容。

### 左侧展示全部羁绊列表

左侧详情区用于展示全部羁绊进度，而不是只展示一个被放大的主羁绊。

列表分为两组：

- 阵营羁绊：蜀、魏、吴、群。
- 职业羁绊：猛将、智将、敏将、辅将。

每一行展示羁绊名称、类别、当前人数进度和激活状态。已激活项可高亮，但不应放大成唯一主卡；未激活项保持中性状态，不使用锁、充值提示或失败感样式。

### 当前选中羁绊详情

左侧下方显示当前选中羁绊的详情：

- 参与武将。
- 关键词标签。
- 2 人效果。
- 4 人扩展效果。

2 人效果用于默认体验，4 人效果作为扩展阵容上限预览，不做付费引导，也不暗示 4 人获得专属奖励池或战备池。

### 中部保留 2 + 2 上阵结构

中部继续使用阵容编成页的 2 + 2 槽位结构：

```text
主将一 / 主将二 / 副将一 / 副将二
```

当前默认 2 人时，前两个槽位展示有效武将，后两个槽位保持安静预留。羁绊页签可以在阵型附近用小标签展示当前摘要，例如“蜀 2/4 已激活”“猛将 1/2”“辅将 1/2”。

### 底部摘要保留

底部保留“羁绊效果预览”摘要条，用短标签展示当前阵容带来的方向，例如：

```text
蜀势 +2 / 猛将 +1 / 辅将 +1
```

底部主操作仍为“保存阵容”，不改成“保存配置”或“确认羁绊”。

## 视觉规范

- 继承 `lineup-formation-v01` 的暗色顶部栏、浅色地图底、细边框和朱红页签高亮。
- 羁绊行使用克制的阵营/职业印章或图标，不做强发光稀有度表现。
- 进度信息应清晰，但不要像数值表格一样密集。
- 未激活羁绊是阵容组合结果，不应呈现为付费锁定。

## 后续出图注意

- 若后续调整阵容编成页结构，羁绊页签必须同步保持同层级切页关系。
- 若新增羁绊或改变阈值，需要同步更新本页的列表结构和 `synergy-design.md`。
- 若制作 4 人位解锁界面，不要把羁绊页签变成商业化入口；4 人效果只能作为中性预览。
