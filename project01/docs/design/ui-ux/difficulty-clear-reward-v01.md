# 难度通关奖励 3 选 1 v01

## 资源

- 预览图：`assets/ui/preview/difficulty-clear-reward-v01.png`
- 状态：已确认
- 界面：难度通关奖励 3 选 1
- 触发时机：当前关卡难度的全部波次完成后

## 界面定位

该界面用于难度通关后的中周期奖励反馈。它不是波次间战备，不负责预览下一波敌人，也不提供继续下一波操作。

玩家在这里完成三件事：

- 确认当前难度已经通关。
- 查看固定局外资源与首通解锁。
- 从 3 个通关奖励候选中选择 1 个。

## 已确认设计规则

### 与波次战备三选一区分

本界面继承“三选一清晰决策”的结构，但语义必须区别于 `battle-prep-choice-v04`：

- 不显示下一波敌人预览。
- 不显示本局战备摘要。
- 不使用“继续下一波”作为主操作。
- 奖励影响装备池、技能强化、战备重掷、局外资源或解锁内容。

### 中央奖励三选一

中央展示 3 张候选奖励卡。每张卡需要清楚表达：

- 奖励名称。
- 奖励类型。
- 关键词。
- 生效范围或带回规则。

当前示例包含：

- 装备奖励：获得玄铁盾。
- 技能强化：张飞：燕人守势强化。
- 战备重掷：再整旗鼓。

同一轮候选不应让 3 张卡都属于同一奖励类型。

### 通关摘要

通关摘要用于帮助玩家回顾本局表现，但不展开完整战报。建议展示：

- 波次进度。
- 用时。
- 阵容。
- 当前流派。

### 固定资源与首通解锁

右侧面板承载通关后的确定性反馈：

- 固定获得资源，例如兵书残页、铜钱。
- 首通解锁，例如新关卡难度、装备池扩展。

这部分不参与三选一，避免玩家误以为固定奖励需要选择。

### 底部操作

主操作为“确认奖励”。辅助操作可以是：

- 查看本局记录。
- 返回地图。

主操作权重必须高于辅助操作。

## 视觉规范

- 可以有通关后的明亮完成感，但不使用页游礼包、抽卡开奖或黑金领奖风格。
- 面板保持浅色、克制、清晰，奖励卡不使用夸张发光宝箱。
- 中文文案短促，避免长段说明。

## 后续出图注意

- 如果做失败结算页，应明确不展示奖励 3 选 1，只给少量资源和失败记录。
- 如果做完整战报页，可以从“查看本局记录”进入，不应塞入本界面主流程。
