- 预览图：`assets/ui/preview/challenge-failure-settlement-v03.png`

- 界面：挑战失败结算
- 触发时机：我方全灭后

## 界面定位

该界面用于失败分支的结算反馈。它与 `difficulty-clear-reward-v01` 共用结算框架，让玩家明确当前已经进入一局结束后的反馈阶段。

它不提供奖励 3 选 1，不展示首通解锁，不进入下一波战斗。

玩家在这里完成三件事：

- 确认失败发生的位置。
- 查看少量失败资源和调整建议。
- 选择再试一次、调整阵容或返回地图。

## 已确认设计规则

### 与通关奖励页保持同源

失败页应与难度通关奖励页保持布局亲缘关系：

- 顶部资源栏一致。
- 背景使用淡化战略地图或山川纹理。
- 左侧为挑战摘要。
- 右侧为结算信息与建议。
- 底部保留左辅助、中主操作、右辅助的结构。

这能让玩家形成统一认知：胜利和失败都是结算，只是中部内容不同。

### 中央不使用三卡结构

失败页中央不使用三张卡，避免误导玩家以为失败后也有 3 选 1 机制。

当前版本使用一张战败静态图承载失败情绪：

- 残旗。
- 破损盾牌。
- 撤退剪影。
- 烟尘与落日。

画面克制，不使用血腥、恐怖或红黑压迫风格。

### 失败摘要

左侧挑战摘要建议展示：

- 波次进度。
- 用时。
- 阵容。
- 状态。

当前示例为：

- 波次 4/6。
- 用时 12:18。
- 阵容 赵云 / 张飞。
- 状态 我方全灭。

### 少量获得

失败后只展示少量局外资源，且应明显低于通关奖励。

当前示例：

- 兵书残页 +3。
- 铜钱 +80。

### 调整建议

右侧建议用于帮助玩家下一局调整，但不展开完整阵容编辑。

当前示例：

- 提高护盾/反击装备。
- 携带治疗或减伤战备。
- 优先处理弩手。

### 底部操作

主操作为“再试一次”。辅助操作为：

- 返回地图。
- 调整阵容。

主操作权重必须高于辅助操作。

## 后续出图注意

- 不要加入奖励 3 选 1、确认奖励、首通解锁或获得装备。
- 不要把失败原因做成三张可选卡。
- 若需要更详细战报，应由额外的“查看战报”或日志页承载，不塞入本页主流程。
