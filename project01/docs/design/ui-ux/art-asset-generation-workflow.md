# 美术资源出图与切片流程规则

本文档记录项目级美术资源生产流程。后续生成 UI 控件、卡牌、按钮、特效序列帧、HUD 元素等游戏美术资源时，默认遵守本文档。

## 核心原则

美术表现优先由图像生成或人工设计方案完成，脚本只负责工程化整理、切片、校验和批量一致性处理。

除非用户明确要求直接产出最终切片，否则不要跳过审美确认阶段。不要用脚本绘制替代主要美术创作。

## 标准三阶段流程

### 1. 氛围图 / 效果图阶段

目标是先解决审美方向。

执行方式：

- 使用图像生成或参考图重绘方式，先产出完整效果图、氛围图或整套 UI 展示图。
- 优先追求整体构图、风格、材质、细节密度、光影、色彩和游戏美术质感。
- 允许图中存在临时文字、占位内容、展示性排版，但需要明确告知用户该阶段不是最终可用切片。
- 产出后必须等待用户审美确认。

禁止事项：

- 不要在用户确认审美前就拆资源。
- 不要因为规格、尺寸、透明背景等工程要求而牺牲第一阶段的美术表现。
- 不要用脚本几何绘制直接冒充最终美术效果图。

### 2. 清稿 / 无文字可拆版本阶段

目标是把已确认的审美方案转成更适合切片的干净图。

该阶段不是每次都需要。如果第一阶段效果图本身没有文字、元素边界清晰、背景可控，可以跳过。

适用场景：

- 效果图里有写死文字，但最终资源不能包含文字。
- 元素之间距离太近，直接切片会互相粘连。
- 背景复杂，透明抠图风险高。
- 用户要求“照着效果图重绘，但不要复制原图像素”。

执行方式：

- 以已确认的效果图为视觉基准，重新生成或重绘无文字、可拆、元素间距更干净的版本。
- 保留已确认的风格、比例、材质和装饰密度。
- 不新增未经确认的视觉方向。
- 完成后再次给用户确认。

禁止事项：

- 不要用粗糙色块覆盖文字。
- 不要从带文字图中硬裁后伪装成干净资源。
- 不要在清稿阶段大幅改变第一阶段已确认的美术风格。

### 3. 工程切片 / 资源校验阶段

目标是把已确认的图整理成项目可用资源。

执行方式：

- 用脚本或图像处理工具完成切片、透明背景、尺寸统一、命名、manifest、预览图和校验。
- 对透明 PNG 做 alpha 边缘检查，确认非矩形控件四周有透明安全边。
- 对矩形背景类资源明确标注其为不透明矩形资源。
- 输出棋盘格预览或组合预览，用于检查透明区域是否干净。
- 修改切片坐标或抠图规则时，只改对应资源，不无故重做已确认资源。

推荐输出：

- `source_*`：已确认的整图或清稿源图。
- `transparent/`：最终透明 PNG 切片。
- `preview_*`：普通预览和透明检查预览。
- `manifest.json`：资源尺寸、来源坐标、九宫格建议、状态说明。
- `README.md`：资源用途、状态和注意事项。

禁止事项：

- 不要让脚本承担主要美术设计工作。
- 不要在最终资源中留下临时文字、背景底色、邻近元素碎片。
- 不要覆盖已通过审核的目录；新版本用独立版本目录保存。

## 用户确认点

默认确认顺序：

1. 用户确认氛围图 / 效果图审美方向。
2. 如需要，用户确认清稿 / 无文字可拆版本。
3. 用户确认最终切片资源。

如果用户只说“继续”，通常表示进入下一阶段，但仍应根据当前阶段说明下一步要做什么。

## 何时可以跳过阶段 2

满足以下条件时可以跳过清稿阶段：

- 效果图没有写死文字，或文字不会进入最终资源。
- 元素间距足够，不会切到邻近元素。
- 背景易于分离，透明区域可控。
- 用户明确表示可以直接切片。

跳过时仍要说明：“当前图满足可直接切片条件，因此进入工程切片阶段。”

## 工具使用边界

图像生成适合：

- 审美探索。
- 整体效果图。
- 高质感 UI 氛围图。
- 材质、光影、装饰密度复杂的图像。

脚本适合：

- 尺寸统一。
- 透明 PNG 输出。
- 切片。
- 状态批量生成。
- 命名和 manifest。
- alpha / 边缘 / 预览校验。

脚本不适合：

- 独立承担主要美术设计。
- 在没有审美确认的情况下直接画最终控件。
- 用低美术表现替代图像生成阶段。

## 当前项目默认结论

本项目三国风 UI、卡牌、按钮、特效资源，默认采用：

**先出氛围图，经用户审美确认；必要时再出无文字可拆清稿，经用户确认；最后再做工程切片和校验。**

后续如无特殊说明，Codex 应主动按此流程执行。
