# 军令卡系统 V1.1 UI/UX 任务说明

> 状态：阶段 A 已完成，阶段 B 待 UI/UX 接手。本文只定义已确认的交互与信息要求，不定义美术风格或资源制作方案。

## 任务目标

为单人 PVE 自动战斗准备四个相连界面：备战军令市场、战斗主动军令 HUD、后台军略图鉴、Set 与新解锁信息展示。

## 已确认规则

- 备战阶段固定展示 3 个军令报价；玩家可购买多张，也可刷新整组报价。
- 主动军令与被动军略必须有清晰类型区分。主动军令进入局内军令库，备战时最多选择 3 张进入下一波 HUD；被动军略进入后台图鉴。
- 普通/精英波显示 2 点军令点，Boss 波显示 3 点。每张主动军令每波最多使用一次，使用后保留卡位但显示本波不可用。
- Set 组件必须显示 Set 名称、进度、已拥有/缺失组件、自动融合结果；根系完成后出现两条二阶路线的“新解锁”来源提示。
- 主动军令只允许一次目标选择。选择“上、中、下路”时，表示战前路归属，不是实时地面区域；目标队长/精英或后排队伍等系统锁定结果需要在释放前高亮预览。
- 飞骑、伏兵等由军令召来的效果为临时战术表现，不能让既有英雄队伍移动或改变站位属性。

## 需要 UI/UX 输出

1. 3 报价市场的购买、刷新、余额、已购移除与新解锁提示状态。
2. 主动军令库到 3 槽 HUD 的备战选择流，以及普通/精英/Boss 的军令点状态。
3. 主动军令的可用、军令点不足、无合法目标、选路、目标预览、预警、已使用和返还状态。
4. 被动军略与自动融合结果的后台图鉴、Set 图谱与继承效果说明。
5. 小兵补员入口的最小交互方案：显示队长、当前小兵数、上限、补员成本与不可补员原因。

## 新增任务：备战市场的购买单卡详情概念图

### 任务目标

为玩家在备战军令市场中查看并购买单张卡牌时，设计一张可独立阅读的详情界面。该界面优先回答四个问题：效果是否值得购买、当前是否买得起、是否推进 Set、购买后是否会自动融合或解锁后续路线。

本任务只设计“购买时展开的单张详情”，不要求同时完成完整三报价市场、HUD、图鉴或弹窗流程。

### 必须展示的信息

**顶部：卡牌身份与购买条件**

- 图标、名称、类型标签：`主动军令` / `被动军略`。
- Set 名称；独立军略明确标为“独立军略”。
- 购买价格、军资图标、玩家当前军资，以及可购买 / 军资不足状态。

**中部：玩家可读的效果**

- 使用正式效果文本，包含已定数值、持续时间、目标范围、触发条件与每波限制；不能只写抽象关键词。
- 主动军令额外显示：消耗 `1` 点军令点、每波最多使用 `1` 次、目标方式（无目标 / 合法单位 / 我方或敌方战前上中下路）。
- 若主动军令有预警、系统自动锁定目标或目标失效返还，详情中明确说明；选路始终指战前路归属，不是实时地面坐标。
- 被动军略额外显示：“购买后本局后续波次持续生效”“不占 HUD 槽位”，并写清生效对象或触发时机。

**底部：构筑与自动融合信息**

- Set 进度，例如 `破军 I 2/3`；用图标清楚显示已拥有组件与缺失组件。
- 自动结果预览：名称、图标和一句可读的结果说明。
- 主动 Set 说明“基础主动自动进阶，保留军令库与 HUD 槽位”；被动 Set 说明“组件效果保留，自动吞噬为后台高阶军略”。
- 若购买本卡会立刻完成 Set，必须将“购买后自动融合结果”提升为醒目主状态，列出继承效果和新增效果。
- 根系完成会解锁二阶路线时，显示将获得报价资格的后续路线名称，例如“解锁：虎贲 II、飞骑 II”。

### 不应展示的信息

- `command_card_id`、市场内部的路线位/探索位/自由位、内部结算公式和程序字段。
- 完整军令图谱、完整 HUD 选卡状态或与当前购买无关的全部卡牌。
- 将价格、数值、军令点、进度或卡牌 ID 写死进图标；这些信息由 UI 文本与状态层承载。

### 概念图覆盖场景

至少用下列四种状态验证信息层级；可使用现有军令图标和临时文字：

1. 普通主动 Set 组件，例如尚未完成的斩将令，展示目标方式、军令点与 Set 进度。
2. 普通被动 Set 组件，例如锋矢军略，展示整局持续生效、不占 HUD 与组件进度。
3. 会在购买后立刻完成 Set 的组件，展示基础主动进阶或后台被动吞噬、继承效果、新增效果及二阶路线解锁。
4. 独立军略，例如体魄军略，展示无 Set 进度时的简洁购买信息。

### 交接边界

- UI/UX 可决定信息布局、层级、折叠方式、提示文案长度和概念图视觉方案。
- 不得修改军令效果、数值、军令点、目标规则、融合条件、市场报价规则或军资经济；发现信息过载或规则难以表达时，记录问题回交主策划确认。

## 新增任务：军令图标资源包 V1

### 任务目标

为军令市场、战斗 HUD 和军略图鉴建立一套可扩展的图标视觉语言。图标表达军令效果与 Set 身份，卡牌名称、数值、冷却和进度由 UI 文本承载，不写死进图标。

### 范围与优先级

本批最终覆盖 `42` 个图标：`12` 张主动军令、`30` 张被动军略/自动融合结果。按两阶段推进，未经项目负责人审美确认不得进入切片阶段。

**阶段 A：12 张主动军令，已完成**

- 斩将令、破阵令、奇袭令、奔袭令。
- 焚营令、火攻令、连环令、焚舟令。
- 伏兵令、十面埋伏令、鼓舞令、固守令。
- 已交付 `assets/ui/command-cards/v01/`：12 张 `512 x 512` 无文字透明主图、源图、48px 预览、alpha 预览和资源 manifest。
- 12 个主动军令的 `command_card_id` 与透明主图已核对无缺失；不得重画、替换或覆盖该批资源，除非项目负责人另行提出返修。
- 后续图标必须延续该批的清晰动作轮廓、克制历史策略感与小尺寸辨识度；同一进阶链保留可识别的核心意象，但高阶图标不能只靠换色区分。

**阶段 B：补齐剩余 30 张被动军略与自动融合结果，当前任务**

- 破军/虎贲/飞骑被动：锋矢、战鼓、铁甲、盾列、死守、固阵、良马、侧击。
- 火计/连环/伏兵被动：火油、风助、缚舟、断索、耳目、断粮。
- 屯田/募兵/军械被动：屯垦、节用、缴获、富国、募卒、练卒、成阵、精兵强军、精锻、甲胄、校准、百工。
- 独立被动：体魄、锐气、行军、保全。
- 先输出覆盖全部 30 张的效果图总览，可使用临时标签帮助审核；重点验证被动、融合与高阶结果的意象区分。
- 效果图审核后，再输出无文字透明主图、尺寸统一切片、48px 预览、alpha 预览，并扩展现有 `manifest.json` 与 `README.md`。
- 最终资源放入既有 `assets/ui/command-cards/v01/` 的 `source/`、`transparent/`、`preview/` 目录；不得修改已完成的 12 张主动军令文件。

### 视觉与信息边界

- 整体方向：清晰、轻量、克制的历史策略感；不做页游大厅感、黑金充值感、厚重卷轴、过暗战斗 UI 或过度装饰边框。
- 主动军令以动作、方向、战术事件为主；被动军略以稳定的军略符号、军备、阵法或经营意象为主；自动融合结果需要表现为组件意象的明确升级。
- 可为破军、火计、屯田三条根系建立可辨识的共享母题，但不能只靠颜色表达 Set 归属。
- 图标内禁止固定业务文字、数字、卡牌 ID、冷却、费用、进度或完整卡牌底框。
- 不得把飞骑、伏兵画成现有英雄离开小队执行军令；它们是军令召来的临时战术效果。

### 工程交付要求

- 每个图标先提供 `512 x 512` 的无文字透明主图，主体四周保留约 `12%` 透明安全边。
- 以 `command_card_id` 命名，具体 ID 见 [军令效果设计清单 V1](../tactical-command-card-catalog-v1.md) 的“配置 ID”表。
- 建议新建独立版本目录：`assets/ui/command-cards/v01/`，包含 `source/`、`transparent/`、`preview/`、`manifest.json` 和 `README.md`；不得覆盖已确认的 P0 组件目录。
- 至少提供 `48 x 48` HUD 观察尺度的可读性预览，以及棋盘格 alpha 预览；非矩形图标边缘必须干净。
- 图标风格确认与最终切片必须遵循 [美术资源出图与切片流程](art-asset-generation-workflow.md) 的“效果图确认 -> 无文字清稿 -> 工程切片”顺序。

### 验收标准

- 玩家在不阅读文字时，能够区分攻击突袭、火攻扩散、伏击封锁、鼓舞、护守、经济、补员与军械等大类意图。
- 同一 Set 的三个组件有家族感，跨 Set 不会在小尺寸下混淆；高阶结果在 `48 x 48` 预览中仍可区分于基础组件。
- 无固定业务文字、无邻近元素残片、透明边缘干净；图标可独立嵌入市场卡、HUD 和图鉴。
- UI/UX 只提出信息层级和资源需求，不修改军令效果、数值、Set、目标规则或军令点。

## 非本任务范围

- 不决定卡牌效果、数值、军令点、军资经济和市场权重。
- 不在未经审美确认前直接产出最终切片；资源生产严格按本任务中的两阶段与项目美术工作流执行。
- 不重新引入固定三张通用军令或英雄技能卡池。
