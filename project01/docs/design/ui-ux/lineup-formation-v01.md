# 阵容编成 v01

## 资源

- 预览图：`assets/ui/preview/lineup-formation-v01.png`
- 状态：已确认
- 界面：阵容编成
- 入口：局外主界面 / 地图选关页的“调整阵容”
- 设计方向：承接主界面 v02 的现代历史策略 UI

## 界面定位

阵容编成是局外战前配置界面，不属于局内战斗备战阶段。它用于玩家在开始挑战前调整本次默认出战阵容，并查看基础羁绊与角色配置状态。

该界面不负责完整技能装配、完整装备配置、武将养成或局内 Roguelike 战备选择。它只提供这些系统的轻量入口或摘要，避免界面职责过重。

## 已确认设计规则

### 与主界面顶部栏一致

阵容编成从主界面进入，因此顶部栏必须继承主界面的资源与功能边界：

- 保留：`铜钱 1280`、`兵书残页 6`、设置图标。
- 不新增：体力、玉璧、充值货币、资源加号、商城、礼包、活动入口。
- 返回入口只保留一个，避免顶部和底部同时出现返回按钮。

### 局外入口而非局内备战

该界面用于开始挑战前配置阵容。局内战斗备战阶段最多显示“查看阵容”或“队伍摘要”，不开放完整阵容编成。

这样可以保持局内节奏聚焦于下一波预览和战备三选一，避免每波之间反复换人导致 Roguelike 决策边界混乱。

### 出战结构为 2 + 2

阵容槽位固定按最多 4 人设计：

```text
主将一 / 主将二 / 副将一 / 副将二
```

当前默认 2 人时，前两个槽位展示有效武将，后两个槽位保留安静的预留空间。预留位不应做成强锁定提示、充值提示或奖励提示。

### 主操作唯一

底部主操作为“保存阵容”。不要同时出现另一个同权重的“开始挑战”或“返回地图”按钮。

保存后回到主界面 / 地图选关页，由关卡信息面板中的“继续挑战”承担开战主操作。

### 武将列表不做卡池感

右侧武将列表用于选择可上阵武将，不是抽卡或武将池展示。

应避免：

- 星级大面积展示。
- `+` 号、抽卡感按钮、获得提示。
- 商城或礼包联想。

可以保留：

- 武将头像。
- 名称。
- 阵营。
- 职业或定位。
- 可用状态。

### 技能装配只做轻量入口

阵容编成页中的角色详情不展开完整 `6 选 4` 技能配置。此处只表达当前状态：

```text
技能 4/6
[技能1] [技能2] [技能3] [技能4]  调整技能 >
```

完整技能配置应放在“武将配置”界面的技能页签中处理：

- 候选技能 6 个。
- 当前装配 4 格。
- 点击候选技能替换已装配技能。
- 展示关键词、冷却、技能类型与适配提示。

装备配置与技能配置共用同一“武将配置”界面，通过 `技能 4/6` / `装备 2/2` 页签切换，避免两个高度相似的页面重复出现。

## 视觉规范

- 背景和面板继续使用主界面的清爽战略地图气质。
- 面板以浅色石瓷/宣纸感为主，边框细，不使用厚重卷轴或黑金装饰。
- 羁绊摘要可以放在底部，但要紧凑，不抢主阵容区域。
- 角色详情、阵容槽位和武将列表三者应层级明确，不互相争夺主视觉。

## 后续出图注意

- 做“武将配置”界面时，应继承本页的 `技能 4/6` 和 `装备 2/2` 入口关系，通过页签展开完整技能和装备配置。
- 做“武将养成”界面时，不要把阵容槽位和出战保存逻辑重复搬过去。
