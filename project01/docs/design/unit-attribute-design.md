# Unit Attribute Design

本文档定义《三国斗阵》MVP 中单位、技能、装备和状态会使用的数值属性口径。它只定义“有哪些属性、属性影响什么、首版是否启用”，不规定最终数值公式和 Y3 工程实现。

## 设计目标

- 统一武将、敌人、召唤物和 Boss 的基础属性口径。
- 避免技能、装备、战备、羁绊各自发明属性名。
- 让 Y3 落表时知道哪些字段必须有，哪些字段暂缓。
- 控制 MVP 数值复杂度，不在首版引入暴击、闪避、命中等额外随机系统。
- 保证 UI 展示的属性与实际战斗属性一致。

## 属性分层

属性分为 5 层：

| 层级 | 用途 |
| --- | --- |
| 基础单位属性 | 决定单位最基本的生存、输出和移动能力。 |
| 战斗节奏属性 | 决定普攻、技能释放和状态持续的节奏。 |
| 站位属性 | 决定单位在 3 路 3x3 战场中的战前格位、行列、优先选敌和站位被动。 |
| 关键词状态属性 | 承载流血、灼烧、护盾、破甲等构筑关键词。 |
| 配置与测试属性 | 用于调试、推荐、验收和测试记录，不直接等同战斗数值。 |

## 基础单位属性

| 属性 | 建议字段 | 作用 | 适用对象 | MVP |
| --- | --- | --- | --- | --- |
| 生命 | `base_hp` | 单位承伤池，生命归零后死亡。 | 武将、敌人、召唤物、Boss | 必做 |
| 攻击 | `base_attack` | 普攻和部分伤害技能的基础来源。 | 武将、敌人、召唤物、Boss | 必做 |
| 防御/护甲 | `base_armor` / `armor` | 降低受到的伤害，或作为高护甲、破甲相关判断。 | 武将、敌人、Boss | 必做 |
| 攻击距离 | `attack_range` | 单位可进行普攻的距离。 | 武将、敌人、召唤物、Boss | 必做 |
| 移动速度 | `move_speed` | 单位基础移动能力；用于进入战斗、短距离调整、追击或归位。 | 武将、敌人、召唤物 | 必做 |
| 战斗移动速度 | `combat_move_speed` | 接敌后短距离调整、追击、突进或归位速度。 | 武将、敌人、召唤物 | 必做 |
| 攻击间隔 | `attack_interval` | 普攻间隔，数值越低普攻越频繁。UI 可展示为攻速。 | 武将、敌人、召唤物、Boss | 必做 |

## 战斗节奏属性

| 属性 | 建议字段 | 作用 | 适用对象 | MVP |
| --- | --- | --- | --- | --- |
| 技能冷却 | `cooldown` | 技能释放后再次可释放所需时间。 | 技能 | 必做 |
| 释放距离 | `cast_range` | 技能可释放距离。 | 技能 | 必做 |
| 目标规则 | `target_rule` | 决定技能或攻击优先选择谁。 | 技能、特殊普攻 | 必做 |
| 效果值 | `effect_value` | 技能、装备或战备的主要效果量。 | 技能、装备、战备、羁绊 | 必做 |
| 持续时间 | `duration` | 状态、增益、减益、控制的持续时长。 | 技能、状态、装备效果 | 必做 |
| 最大层数 | `max_stack` | 可叠加状态的层数上限。 | 流血、灼烧等状态 | 必做 |
| 内置冷却 | `internal_cooldown` | 限制高频触发效果，避免无限循环。 | 装备、羁绊、战备、被动技能 | 必做 |
| AI 优先级 | `ai_priority` | 自动释放技能时的优先级参考。 | 技能 | 建议 |
| 是否主动技能 | `is_active_skill` | 决定技能是否可被自动释放或进入手动技能卡牌池。 | 技能 | 必做 |
| 默认自动释放 | `default_auto_cast` | 已上阵英雄主动技能的默认自动释放状态。 | 技能/装配 | 必做 |
| 是否可进卡池 | `can_enter_card_pool` | 决定关闭自动释放后，该技能是否可进入 HUD 技能卡牌池。 | 技能 | 必做 |

## 站位属性

| 属性 | 建议字段 | 作用 | 适用对象 | MVP |
| --- | --- | --- | --- | --- |
| 战前格位 | `battle_grid_slot` | 1-9 的站位格编号。我方布局为 `741 / 852 / 963`，敌方镜像。 | 武将、敌人 | 必做 |
| 所在行 | `battle_lane` | 上、中、下三路，可由格位推导，也可为调试字段。 | 武将、敌人 | 必做 |
| 所在列 | `battle_column` | 前、中、后三列，可由格位推导，也可为调试字段。 | 武将、敌人 | 建议 |
| 优先选敌组 | `primary_target_slots` | 当前格位或当前行优先攻击的敌方格位组。 | 武将、敌人 | 必做 |
| 次级选敌组 | `fallback_target_slots` | 优先格位无目标时的候选敌方格位组。 | 武将、敌人、技能 | 必做 |
| 站位被动条件 | `position_passive_rule` | 该英雄或技能在指定格位、行、列触发的被动规则。 | 武将、技能 | 建议 |
| 存活上限 | `max_alive_limit` | 控制单波或召唤物的同屏数量。 | 波次、召唤物 | 必做 |

## 关键词状态属性

| 关键词 | 类型 | 效果口径 | MVP |
| --- | --- | --- | --- |
| 流血 | 持续伤害 | 造成持续物理伤害，可与斩杀、低血目标、连击联动。 | 必做 |
| 灼烧 | 持续伤害 | 造成持续法术伤害，可与范围、扩散、清杂联动。 | 必做 |
| 护盾 | 防御 | 优先吸收伤害，可与反击、前排承伤联动。 | 必做 |
| 反击 | 触发攻击 | 受击或护盾破裂后触发额外攻击。 | 必做 |
| 破甲 | 减益 | 降低目标防御或提高其受到伤害。 | 必做 |
| 连击 | 攻击节奏 | 造成多次命中或追加攻击。 | 必做 |
| 士气 | 资源/增益 | 提供队伍节奏收益，可影响技能循环、攻速、攻击或治疗。 | 必做 |
| 治疗 | 恢复 | 恢复生命或提供持续恢复。 | 必做 |
| 眩晕 | 控制 | 短时间无法移动、攻击和释放技能。 | 必做 |
| 减益 | 泛用削弱 | 降低敌方攻击、防御、攻速或治疗等能力。 | 建议 |
| 召唤 | 单位生成 | 生成额外单位，受同屏单位上限和波次清理规则约束。 | 建议 |

## 展示属性与隐藏属性

玩家需要看到的属性：

| 属性 | 展示建议 |
| --- | --- |
| 生命 | 血条或生命摘要。 |
| 攻击 | 阵容/技能装配页简短展示。 |
| 防御/护甲 | 阵容/技能装配页简短展示。 |
| 攻速 | 用 UI 展示名，不直接展示攻击间隔。 |
| 移速 | 只在阵容或角色摘要中轻量展示。 |
| 技能冷却 | 技能卡必须展示。 |
| 关键词 | 用图标、标签和状态表现展示。 |

默认隐藏或仅策划可见：

| 属性 | 原因 |
| --- | --- |
| 战斗移动速度 | 玩家不需要区分行军和接敌后的移动参数。 |
| 所在行/列 | 可由格位推导，普通玩家不一定需要直接看到。 |
| 优先/次级选敌组 | 属于选敌规则和调试字段，UI 可用更直观的目标线或标签表达。 |
| 内置冷却 | 过细，容易增加 UI 负担。 |
| AI 优先级 | 自动释放调试字段。 |
| 敌人完整数值 | 预览只展示威胁标签，不展示完整数值。 |

## MVP 暂缓属性

首版不建议加入以下属性：

| 属性 | 暂缓原因 |
| --- | --- |
| 暴击率 | 会引入额外随机波动，干扰首轮流派验证。 |
| 暴击伤害 | 需要配套暴击系统，暂缓。 |
| 命中率 | 自动战斗中容易造成不可解释的失败。 |
| 闪避率 | 增加随机性，影响战斗可读性。 |
| 格挡率 | 与护盾、反击、防御功能重叠。 |
| 吸血 | 容易和治疗、护盾、反击形成失控续航。 |
| 元素抗性 | 会让灼烧等关键词过早复杂化。 |
| 控制抗性 | 首版控制压力较弱，暂不作为基础属性。 |
| 资源获取倍率 | 士气已经承担资源节奏，暂不额外拆复杂倍率。 |

## 设计边界

- 属性命名以 Y3 表字段为准，UI 可以使用更易懂的展示名。
- 技能、装备、战备、羁绊优先修改关键词和战斗节奏，不鼓励只堆攻击、防御、生命。
- 同一效果只能归入一个主属性口径，避免“护甲、防御、减伤、抗性”混用。
- 如果后续新增属性，必须先确认它是否真的解决设计问题，而不是增加数值复杂度。
