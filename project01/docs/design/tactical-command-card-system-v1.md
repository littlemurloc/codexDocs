# 军令卡系统 V1

> 状态：已确认的规则框架。替代固定三张通用军令的 V0；具体卡牌、数值、属性换算与经济参数尚待设计。

## 设计定位

英雄的已解锁主动技能始终自动释放，保证英雄自身的稳定战斗循环。军令卡系统承担局内构筑、备战取舍与战斗中的主动介入，不再把英雄主动技能改造成手动卡牌。

军令卡的核心不是“每波三个按钮”，而是让玩家在单局内通过购买、积累、合成与解锁，逐步形成一套会影响后续波次的策略组合。

## 已确认边界

- 所有军令卡、军资、解锁进度、合成结果与属性收益均只在本局内存在；本局结束后全部清空。
- 英雄已解锁主动技能始终自动释放，不提供按英雄关闭自动释放的开关，也不进入军令卡池。
- 军令卡只分为主动军令与被动军略两种类型。
- “进阶军令”不是第三种卡牌类型，而是主动军令或被动军略通过升级、合成或 Set 解锁所达到的高阶形态。
- 卡牌的类型、获得方式和展示位置必须分开定义，不能因为一张牌在后台展示，就把它误判为被动或不可购买。

## 局内来源：备战军令市场

- 每次进入备战阶段，军令市场展示 3 个报价。
- 玩家使用局内货币购买报价中的军令卡；局内货币暂称“军资”，正式名称和产出规则待定。
- 只要军资足够，玩家在同一次备战中可以购买多张卡，不受“每次备战最多购买 1 张”的限制。
- 玩家可消耗军资刷新当前 3 个报价。刷新价格、递增规则、每波刷新上限与购买价格均待后续设计。
- 已购买的卡从本次报价中移除；是否补位及补位规则待后续交互和经济设计确认。
- 报价候选来自当前已具备资格的军令池，而不是固定三张卡或战斗中的随机抽手牌。

## 卡牌类型与展示

### 主动军令

- 购买后进入战斗 HUD，由玩家主动释放。
- 主要承担可感知的战场事件、局势扭转或明确的战术时机，而非单纯的短时数值按钮。
- 单张主动军令至多允许一次手动目标选择；可无目标，也可按卡牌规则选择一个合法目标或一个合法区域。
- 不设计“先选择一名友方、再选择一名敌方”这类两次选目标操作。
- HUD 槽位、携带上限、单波使用次数、冷却或充能规则，待具体卡牌目录与战斗节奏一并确定。

### 被动军略

- 购买后在本局剩余全部波次持续生效，不以短时 Buff 作为默认形态。
- 被动收益可以作用于二级属性、一级战斗属性、局内经济、英雄经验与等级、技能等级，或其他明确的局内战斗规则。
- 其中力量、智力、敏捷与攻击、防御、生命、攻速、移速、闪避、暴击等属性的换算关系尚未定稿；不得在具体卡牌中擅自假定固定换算数值。
- 被动军略不占用主动军令 HUD 槽位，应收纳在局内的军略图鉴或等价后台界面中供玩家查看。

## 吞噬、合成与 Set

### 被动吞噬

- 多张被动军略满足指定 Set 或合成条件时，可被吞噬并转化为一个高阶被动军略。
- 合成后以高阶被动军略展示在军略图鉴中，避免被动卡无限堆叠在前台。
- 合成必须让玩家理解为“升级”，不能产生已购买收益无故消失的体验；具体继承、替换与新增收益逐张卡定义。

### Set 解锁

- 高阶主动军令与高阶被动军略可以在局初不进入报价候选池。
- 当玩家收集并满足某个 Set 的主动或被动军令条件后，对应高阶卡牌才获得进入后续报价候选池的资格。
- Set 解锁不等于免费获得卡牌；已解锁的高阶卡仍需在报价中出现并由玩家使用军资购买。
- Set 完成状态必须独立记录。被动军略被吞噬后，不得丢失已经达成的 Set 或解锁资格。
- 高阶卡解锁后的出现权重、保底方式与实际价格待后续经济设计确认。

## 设计约束

- 主动军令应制造可观察、可判断时机的战场事件，不能大量退化为“若干秒内加/减 X%”的同质化效果。
- 被动军略可支持属性、经济、成长、兵种、站位或战场规则等不同流派，不限定为临时 Buff。
- 经济可以成为完整的局内构筑方向，但必须同时提供军资来源、消费压力和可转化的战斗收益，避免只做无互动的资源累积。
- 经验、等级、技能等级等成长型收益不得绕过英雄升星解锁技能的既定规则；具体强化范围和上限待定。
- 军令卡不得替代英雄技能的职业身份。英雄的强项来自其自动技能与小队，军令负责全局战术和局内构筑。

## 待后续定案

- 军资的正式名称、来源、结算、初始值和价格体系。
- 三报价的刷新、补位、稀有度、权重、保底与购买成本。
- 主动军令的 HUD 槽位、持有上限、释放次数、冷却或充能规则。
- 力量、智力、敏捷对一级属性的映射，以及防御、移速、暴击等属性的完整战斗公式。
- 英雄等级、经验和技能等级的获得方式、作用边界与上限。
- 首批主动军令、被动军略、进阶形态与 Set 的具体目录、效果和数值。
- 合成后低阶被动收益的继承、替换与可读性规则。
