# Hero Skill Template Guideline

> 当前状态：英雄技能设计规范。本文用于后续新增英雄、复盘旧英雄和审查技能稿，不替代当前 10 英雄落表主源。

## 使用范围

本文适用于《三国斗阵》当前版本的英雄技能设计，尤其是以下工作：

- 新增英雄技能初稿。
- 检查英雄 6 个技能是否形成完整循环。
- 判断主动技能的自动释放规则是否清晰、可落表。
- 判断站位、小队、小兵和核心机制是否清晰可读。
- 为后续 Y3 落表和原型验证提供统一口径。

当前英雄技能落表主源以 [first-batch-10-heroes-skill-config-v1.md](first-batch-10-heroes-skill-config-v1.md) 为准；玩家主动参与规则见 [tactical-command-v0.md](tactical-command-v0.md)。

## 新英雄立项前检查

新增英雄进入 6 技能设计前，必须先完成以下检查，避免凭单个英雄灵感直接开写。

### 职业分布检查

当前新英雄挑选需要照顾已确认英雄的类型分布，至少先按“战 / 法 / 牧 / 贼”四类做粗分管理。

- 战：前排、防御、近战压制、正面输出、护卫等。
- 法：法术输出、范围压制、控制、战场规则变化等。
- 牧：治疗、护盾、净化、增益、救场等。
- 贼：侧翼、刺杀、干扰、破阵、卡牌节奏等。

新英雄优先补当前短板职业。若新增已经偏多的职业，必须先说明它和已有英雄的差异化理由。

当前首批 10 英雄的粗分可暂按以下方式观察，后续可再正式归类：

| 职业 | 已确认英雄 | 当前判断 |
| --- | --- | --- |
| 战 | 张飞、夏侯惇、吕布、赵云 | 偏多 |
| 法 | 诸葛亮、司马懿 | 已有基础 |
| 牧 | 刘备 | 偏少 |
| 贼 | 吕蒙、甘宁 | 已有基础 |
| 远程物理输出 | 黄忠 | 暂作独立观察，后续决定归入战、贼或单列射手 |

### 基础战斗类型确认

新英雄设计前，必须向项目负责人确认基础战斗类型和小兵类型。未确认前，不得在技能里擅自绑定武器、攻击距离或兵种。

至少需要确认：

- 英雄基础攻击类型：近战、远程、法术、辅助站桩、混合等。
- 英雄主要职责：输出、承伤、治疗、控制、干扰、增益、召唤等。
- 英雄小兵类型：近战、远程、混合、无特殊要求等。
- 是否允许技能绑定具体武器表现，例如射击、挥砍、法术、号令。

未确认时，技能描述只能使用中性表达，例如“普攻命中”“攻击当前目标”“造成伤害”，不得写“射击”“挥斩”“远程小兵”“近战伤害”等会提前锁死表现和配置的内容。

## 英雄技能标准结构

每名英雄默认拥有 6 个技能：

- 3 个主动技能。
- 3 个被动技能。
- 技能通过外围英雄升星依次解锁。
- 英雄进入战斗时自动携带所有已经解锁的技能。
- 旧版“6 个技能战前携带 4 个”的装配规则已经废弃。

推荐设计顺序为：

| 阶段 | 技能类型 | 设计目标 |
| --- | --- | --- |
| 阶段1主动 | 主动 | 建立英雄最基础的战斗行为。 |
| 阶段1被动 | 被动 | 启动核心机制或基础站位定位。 |
| 阶段2主动 | 主动 | 提供不同于主动1的战术用途。 |
| 阶段2被动 | 被动 | 强化定位、站位收益或小队关系。 |
| 阶段3主动 | 主动 | 提供英雄的高价值战术爆点。 |
| 阶段3被动 | 被动 | 完成核心循环，补足后期辨识度。 |

## 核心机制规范

每名英雄原则上应有 1 个主要专属机制，贯穿其主动和被动。

核心机制需要明确：

- 如何获得。
- 最多几层或是否只有开关状态。
- 是否会被主动技能消耗。
- 什么时候失去。
- 是否跨波次保留。
- 玩家能否从 UI 或战斗表现中看懂当前状态。

核心机制不应只写成纯数值标签。它应该让玩家理解这个英雄的玩法，例如：

- 黄忠：守后排积累蓄势，再用主动技能转化为压制、穿透或斩首。
- 刘备：通过友方队长低血获得仁德，再强化治疗和护盾。
- 赵云：通过闪避获得游龙，再转化为额外攻击或救援护盾。
- 张飞：通过承受近战伤害获得守势，提升生存并强化控制和护盾。

核心机制来源需要多样化，不应默认做成“每 X 秒获得 1 层”。定时资源可以使用，但必须有明确设计理由，并避免多个新英雄连续使用同一类循环。

常见机制来源可包括：

- 战前站位型：根据布阵格决定资源、强化或触发条件。
- 受击承压型：通过承受特定压力获得资源。
- 命中/闪避型：通过攻击命中、闪避、格挡等战斗事件获得资源。
- 击杀/参与击杀型：通过击杀队长、精英或参与击杀获得滚雪球收益。
- 友方状态型：通过友方低血、护盾破裂、治疗目标等触发。
- 敌方状态型：通过敌方低血、被控制、被破防等触发。
- 波次节奏型：通过波次开始、敌方增援、精英出现等事件触发。

## 主动技能规范

三个主动技能不能只是同类技能的数值升级。它们应至少在以下维度中形成差异：

- 目标类型不同：当前目标、一路敌人、低血队长、友方队长等。
- 战术职责不同：单点、穿透、控制、护盾、治疗、斩杀、救援。
- 资源消耗不同：消耗 1 层、消耗全部、不消耗但触发额外状态。
- 自动释放规则不同：目标筛选、释放阈值、冷却时的优先级或资源消耗条件不同。

主动技能统一使用自动释放模式：

- 已解锁主动技能始终自动释放；英雄技能不进入卡牌池，也不提供按英雄关闭自动释放的开关。
- 主动技能的自动目标、释放阈值和优先级必须可落表，且不能因缺少玩家选点而失效。
- 玩家主动参与由固定三张通用军令承担。军令不与英雄技能共用冷却，也不改变英雄技能的自动释放规则，详见 `tactical-command-v0.md`。

## 被动技能规范

被动技能主要承担三类职责：

- 启动核心机制：例如获得层数、定义触发条件。
- 强化英雄定位：例如前排承伤、后排输出、侧翼刺杀。
- 补足小队或站位关系：例如小兵受益、后排防护、侧翼攻速。

被动技能不应堆砌过多独立规则。若一个被动同时包含多个条件，需要确认玩家是否能理解它们之间的关系。

## 站位联动规范

当前站位术语统一如下：

- 前排 = `1/2/3`
- 中排 = `4/5/6`
- 后排 = `7/8/9`
- 侧翼 = `1/4/7` 或 `3/6/9`

站位效果使用“战前布阵锁定”规则：

- 站位只取自战前布阵阶段的 3x3 格子。
- 波次开始后，英雄的站位属性在本波次内固定。
- 战斗中移动、追击、后撤、被击退、绕行或贴近敌人，不会改变该英雄的站位属性。
- 技能中写“位于前排/后排/侧翼”时，设计含义是“本波次开战前布置在对应站位”。
- 下一次备战调整站位后，下一波次重新锁定站位属性。
- 禁止写“离开前排/后排/侧翼后失效”这类动态站位判断。

站位联动应满足：

- 布阵阶段能提示是否触发。
- 战斗中能看出收益是否存在。
- 不要求玩家靠记忆背隐藏规则。
- 不把同一个英雄的核心玩法绑定到过多格子条件。

如果技能因为站位触发，应在布阵 UI 中提供明显提示，例如高亮、标签、预览文本或触发特效。

## 小队与小兵规范

当前英雄小队规则如下：

- 队长 = 英雄。
- 每个英雄带领一支小兵队伍。
- 玩家队长死亡时，该队小兵溃败离场。
- NPC 队长死亡后，该队小兵继续战斗。
- 小兵和队长都是独立个体，都可以被技能选为目标。
- 当前小兵兵种只确认近战和远程。

技能设计需要遵守：

- 写“队长/英雄”时，目标筛选忽略小兵。
- 写“小兵”时，目标筛选忽略队长。
- 不依赖医疗兵等未确认兵种。
- 不设计让小兵频繁切换目标或脱离队长行为的机制。
- 小兵联动应尽量是稳定增益、减伤、伤害修正或兵种收益。

## 玩家阅读版规范

技能玩家阅读版要写正式效果，不写只有设计讨论上下文才能理解的句子。

必须明确：

- 数量：例如“2 次伤害”“最多 3 个敌人”。
- 时间：例如“持续 3 秒”“每 8 秒最多触发一次”。
- 阈值：例如“生命低于 40%”。
- 目标：例如“当前目标”“生命比例最低的友方队长”。
- 消耗：例如“消耗 1 层仁德”“消耗全部威势”。

避免使用：

- “多次伤害”
- “短时间”
- “生命较低”
- “范围内”但没有定义范围
- “该加成提高”但不说明玩家能感知什么

工程限制、目标筛选、内置冷却、系统锁定等内容可以写在落表注释中，不必塞进玩家阅读版。

## 禁用或慎用机制

当前阶段不建议设计以下机制：

- 依赖技能前摇/后摇缩短、延长、取消的机制。
- 让小兵频繁切换目标的机制。
- 判断“正在被某个单位攻击”的复杂条件。
- 依赖未确认兵种的机制，例如医疗兵。
- 过度随机的次级选敌规则。
- 需要玩家实时操作走位才能成立的机制。
- 一个属性反复自我放大的循环，例如“拥有某资源提高闪避，闪避又大量获得该资源”。

敌人读性中的动作前摇提示不属于本条禁用范围；本条主要限制英雄技能机制收益不要绑定前后摇调整。

## 审核清单

新英雄技能稿进入评审前，至少检查以下问题：

- [ ] 是否有清晰的 1 个核心机制。
- [ ] 新英雄是否优先补当前短板职业，或已说明新增偏多职业的差异化理由。
- [ ] 英雄基础攻击类型、小兵类型和职业归属是否已由项目负责人确认。
- [ ] 核心机制来源是否有差异化，未无理由套用“每 X 秒获得 1 层”。
- [ ] 是否是 3 主动 + 3 被动。
- [ ] 三个主动技能是否有不同战术用途。
- [ ] 主动技能的目标筛选、释放阈值、冷却和资源消耗是否可由自动规则完整处理。
- [ ] 被动技能是否服务定位、站位或核心机制。
- [ ] 站位收益是否能在布阵或战斗中被玩家看懂。
- [ ] 小队/小兵联动是否不破坏队长跟随规则。
- [ ] 是否没有使用废弃的 6 选 4、旧装备、旧战备、旧羁绊口径。
- [ ] 玩家阅读版是否明确了数量、时间、目标和阈值。
- [ ] 是否没有引入当前禁用或慎用机制。
