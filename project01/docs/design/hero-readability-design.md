# Hero Readability And Skill Tell Design

本文件用于定义首批 16 名 MVP 武将的战斗表现读性、技能前摇和关键词状态表达。它服务于后续美术、特效、音效、UI 提示和 Y3 内实际配置，不包含代码实现。

## 设计目标

- 玩家在自动战斗中能快速判断谁是前排、谁是输出、谁是辅将。
- 玩家能看懂关键技能即将释放，而不是只在伤害数字出现后才理解结果。
- 同一关键词在不同武将身上保持一致颜色和状态语言。
- 2 人参战时信息不过载，4 人参战时仍能看清核心角色正在做什么。

## 通用表现规则

职业轮廓：

- 猛将：站姿宽、动作重、攻击前摇明显，强调承伤、护盾、反击和破甲。
- 智将：施法姿态清楚，技能释放前有阵图、地面范围或元素聚集提示。
- 敏将：移动和攻击节奏快，使用残影、冲刺线、弹道轨迹表达连击和斩杀。
- 辅将：动作偏指挥、旗帜、乐舞或光环，强调治疗、士气、召唤和减益。

关键词颜色建议：

| 关键词 | 主表现 | 读性要求 |
| --- | --- | --- |
| 护盾 | 青蓝色外圈或半透明盾面 | 要能区分临时护盾和角色本体生命 |
| 反击 | 受击后金红闪光、武器回摆 | 必须让玩家知道反击来自被打后的触发 |
| 破甲 | 目标身上裂纹、碎甲音效 | 裂纹不应遮住流血或灼烧状态 |
| 灼烧 | 橙红火苗、地面余火 | 持续伤害和扩散瞬间要有不同强度 |
| 流血 | 暗红切痕、滴落粒子 | 不做大面积红雾，避免和低血提示混淆 |
| 连击 | 白色或银色残影、连续命中点 | 追加命中要有轻量反馈，不能像大招一样抢画面 |
| 斩杀 | 目标低血锁定标记、收束光线 | 不能误导玩家以为目标必死，除非技能确实触发斩杀 |
| 士气 | 金色上升箭头、旗帜波纹 | 队伍增益要显示来源角色 |
| 治疗 | 绿色回流光、生命数字 | 治疗和护盾要用不同颜色区分 |
| 召唤 | 落点标记、短暂入场烟尘 | 召唤物要有明显归属阵营提示 |
| 眩晕 | 头顶晕环、动作停顿 | 控制结束前 0.5 秒可弱化提示，帮助玩家预判恢复 |
| 减益 | 紫黑色符印或暗色向下箭头 | 和灼烧、流血分层显示，避免状态图标拥挤 |

技能前摇分级：

- 轻技能：0.2-0.5 秒前摇，用武器亮光、短线条或头顶小图标提示。
- 中技能：0.6-1.0 秒前摇，用明显动作蓄势、目标标记或小范围地面提示。
- 强技能：1.0 秒以上前摇，用角色姿态、地面范围、音效和屏幕边缘提示共同表达。

## 2 人与 4 人战斗读性

2 人参战：

- 每名武将的核心前摇都可以更完整播放，帮助玩家学习技能逻辑。
- 推荐只强化主角身上的关键技能提示，不把所有普通技能都做成强表现。
- 适合突出“谁在承伤、谁在输出、谁在续航”的关系。

4 人参战：

- 强技能前摇保持完整，轻技能前摇可缩短或弱化。
- 队伍增益和群体治疗必须显示来源角色，但不要为每个受益目标播放完整大特效。
- 同类关键词叠加时，优先显示最高优先级状态：眩晕 > 斩杀锁定 > 护盾破裂 > 灼烧/流血持续。

## 装备表现联动

装备不需要每件都有独立大特效，优先放大角色当前关键词：

- 护盾反击装备：强化护盾外圈、反击回刀、护盾破裂瞬间。
- 灼烧扩散装备：强化火链跳转、火势扩散边界、灼烧层数提示。
- 流血斩杀装备：强化切痕、低血锁定、追加命中反馈。
- 治疗与士气装备：强化来源角色身上的旗帜、光环或号令动作。

装备触发不能比角色技能本体更抢眼。玩家应先看懂“哪个武将用了哪个技能”，再看懂“装备让这个技能更强”。

## 验收标准

- 每名 MVP 武将至少有一个能被玩家记住的轮廓、动作或颜色特征。
- 每名 MVP 武将至少 2 个核心技能有明确前摇或目标提示。
- 玩家能在不看详细数值的情况下分辨护盾、反击、破甲、灼烧、流血、连击、治疗、士气、召唤、眩晕和减益。
- 4 人战斗中，强技能释放前玩家能预判大致影响范围或目标。
- 装备强化效果不会遮盖角色技能本体，也不会让普通攻击表现像强技能。
