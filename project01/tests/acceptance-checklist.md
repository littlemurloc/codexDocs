# MVP Design Acceptance Checklist

验收时建议配合 [design-test-record-template.md](design-test-record-template.md) 记录单局结果、专项问题和调整建议。

## Y3 首轮原型验收用例

本节用于首轮 Y3 原型实装后的主策划验收。它不要求一次性覆盖完整 MVP 内容，重点确认核心流程、规则边界和构筑差异没有偏离设计案。

| 分组 | 用例 | 触发条件 | 期望表现 | 允许简化 | 不可偏离点 |
| --- | --- | --- | --- | --- | --- |
| 站位与选敌 | 3 路 3x3 站位 | 开始任意普通波 | 我方左侧 9 格和敌方右侧镜像 9 格正确生成，玩家可在我方 9 格中摆放队伍。 | 首轮可用简化棋盘格或调试格。 | 不能继续按唯一主路线或旧阵型锚点验收。 |
| 站位与选敌 | 行优先选敌 | 上中下三排分别放入单位并开战 | 上排优先打 `147`，中排优先打 `258`，下排优先打 `369`。 | 命中特效可简化。 | 对应行有敌人时不能随机跨行优先攻击。 |
| 站位与选敌 | 次级选敌 | 对应行无敌方单位时测试 | 上排按 `258 -> 369`，中排优先支援敌方单位更多的一路，数量相同再随机，下排按 `258 -> 147`。 | 中路判断可用日志验证。 | 无目标时不能卡住或无限索敌；中路不能完全无规则随机。 |
| 站位与选敌 | 站位被动 | 将带站位被动的英雄放入触发格与非触发格对照 | 指定格位、行或列能正确触发或不触发被动。 | 首轮可用 1-2 个代理被动验证。 | 被动触发条件必须可解释，不应隐式改变全队规则。 |
| 路径与阵型 | 胜负结束 | 一方单位全灭 | 敌方全灭判定波次胜利，我方全灭判定挑战失败。 | 调试阶段可用日志或简单结算 UI。 | 不因残留死亡单位、卡路径或无限索敌导致战斗无法结束。 |
| 波次状态机 | 普通波胜利进入备战 | 当前波敌人全灭且仍有下一波 | 进入备战状态，展示下一波预览和战备 3 选 1。 | 动画、镜头停留和过场可简化。 | 波次后不能直接给通关奖励，也不能直接开始下一波。 |
| 波次状态机 | 最后一波胜利进入难度结算 | 当前难度最后一波敌人全灭 | 判定该难度通关，处理首通、解锁和通关奖励。 | 首轮可先用简化奖励面板。 | 完成全部波次才算难度通关。 |
| 波次状态机 | 失败结算 | 我方全灭 | 进入失败结算，不展示通关奖励 3 选 1。 | 局外资源可先用固定值或调试值。 | 失败不能触发难度通关、首通解锁或完整通关奖励。 |
| 战备 | 战备 3 选 1 | 普通波或精英波胜利后 | 随机展示 3 个战备，玩家选择 1 个，本次挑战内继承。 | 首轮可先接入较小战备子集。 | 战备不是买武将、换装备或自由改技能的默认入口。 |
| 战备 | 战备继承与清空 | 同一挑战连续进入多次备战，再新开一局 | 已选战备在当前挑战持续生效，新开一局后清空。 | UI 可先只显示文字列表。 | 上一局战备不能污染下一局。 |
| 战备 | 高影响战备专项 | 分别测试破盾回击、火势牵连、弱点齐射、技能预热、破釜沉舟、重新侦察 | 战备效果与设计定位一致，有收益但不造成无脑翻盘或 Boss 跳阶段。 | 可先用代理公式或调试触发。 | 破盾、扩散、弱点、低血类效果必须有触发限制。 |
| 奖励 | 波次后无通关奖励 | 任意非最终波胜利 | 只进入备战，不出现装备奖励、技能强化或局外资源结算。 | 备战 UI 可简化。 | 波次成长由战备承担，难度通关成长由奖励承担。 |
| 奖励 | 难度通关奖励 | 当前难度全部波次完成 | 展示通关奖励 3 选 1，并处理关卡/难度解锁。 | 奖励池可先接入基础子集。 | 默认 2 人位和 4 人位使用同一奖励池。 |
| 奖励 | 门槛解锁 | 关卡 1 难度 5、关卡 2 难度 5、关卡 3 难度 7 通关 | 分别解锁下一关卡难度 1。 | 首轮可用调试按钮辅助抵达门槛。 | 解锁新关卡时不能自动解锁该新关卡后续难度。 |
| 2 人/4 人 | 2 人基础闭环 | 默认 2 人位挑战关卡 1 难度 5 | 至少能体验护盾反击、灼烧扩散、流血斩杀中的基础闭环。 | 不要求所有 2 人组合都通 Boss。 | 2 人位不能变成只能试玩、无法推进基础难度的状态。 |
| 2 人/4 人 | 4 人扩展 | 开启 4 人位后进行同一关卡对照 | 4 人提供羁绊、职责互补和容错提升。 | 付费入口和展示文案可后置。 | 4 人位不获得专属奖励池、战备池、武将池或更高稀有度。 |
| 2 人/4 人 | 门槛难度对照 | 分别用 2 人和 4 人测试关卡 1-3 门槛难度 | 4 人更稳，2 人仍有合理通关路线或明确调优目标。 | 可先只记录代表队伍。 | 关卡 1-3 门槛难度不能被调成必须 4 人。 |
| Boss | Boss 可读性 | 进入任意 Boss 波 | Boss 有关键技能前摇、阶段变化或弱点窗口提示。 | 特效资源可简化为占位表现。 | Boss 不能完全免疫任一核心流派的全部关键词。 |
| Boss | Boss 阶段防跳 | 使用流血斩杀、控制、高爆发队伍挑战 Boss | 能观察到关键阶段或窗口，不被异常直接跳过。 | 可先使用锁血、阶段标记或调试日志验证。 | 斩杀、连击、弱点齐射等不能绕过全部 Boss 表现。 |
| Boss | 失败原因可解释 | 记录通过和失败的代表队伍 | 失败可归因于缺续航、缺破甲、缺前排、缺清杂或缺单体压制。 | 首轮只需记录核心队伍。 | 不能出现无论构筑如何都随机暴毙或无意义拖平的 Boss。 |

### 首轮原型最低通过口径

- 3 路 3x3 站位、选敌、技能卡牌、波次、备战、奖励和结算状态可以完整跑通一局。
- 至少 8 名首批武将、12 件首发装备和一个战备子集可进入实机验证。
- 默认 2 人位能完成关卡 1 基础难度，并能对关卡 1 难度 5 给出测试记录。
- 4 人位只体现阵容宽度和容错，不改变奖励池、战备池或武将获取池。
- 至少保留 3 个代表构筑测试记录：护盾反击、灼烧扩散、流血斩杀。
- Boss 不出现阶段被完全跳过、无限拖战、路径卡死或胜负不结算。

### 与模拟器基线的对照口径

- 模拟器中 `张飞+孙尚香`、`关羽+孙尚香`、`张飞+周瑜`、`夏侯惇+曹操`、`关羽+吕布` 能提供代表通过参照。
- 模拟器中 `吕布+曹操`、`诸葛亮+周瑜`、`张飞+夏侯惇` 的 Boss 失败不直接视为设计错误；Y3 实机应优先判断失败原因是否可解释。
- 若 Y3 实机结果与模拟器明显冲突，先记录实机现象，再判断是 Y3 数值、技能实现、敌人压力、装备代理或设计口径需要调整。

## 核心流程

- [ ] 玩家可以开始新一局 PVE 爬层。
- [ ] 玩家可以调整上阵阵容。
- [ ] 玩家可以进入战斗。
- [ ] 当前波次战斗结束后进入备战状态。
- [ ] 备战状态能预览下一波敌人队伍信息。
- [ ] 备战状态弹出 3 选 1 战备抉择。
- [ ] 玩家每次从 3 个随机战备能力中选择 1 个。
- [ ] 已选择战备能力在本次挑战中持续继承。
- [ ] 新开一局时已选择战备能力清空。
- [ ] 战备抉择后默认允许调整 3x3 站位布局和已上阵英雄主动技能自动释放开关。
- [ ] 波次间调整技能装配或装备必须由特定战备能力或奖励授权。
- [ ] 战斗胜利后进入当前难度的下一波次备战或难度结算。
- [ ] 战斗失败后进入失败结算。
- [ ] 完成某难度全部波次后，该难度通关。
- [ ] 同一关卡内通关当前难度后，下一难度解锁。
- [ ] 完成上一关卡的指定门槛难度后，新关卡难度 1 解锁。

## 3 路 3x3 站位战斗

- [ ] 战场分为上、中、下 3 路。
- [ ] 我方在左侧拥有 3x3 共 9 个战前站位格。
- [ ] 敌方在右侧拥有 3x3 共 9 个镜像站位格。
- [ ] 我方格位编号符合 `741 / 852 / 963`。
- [ ] 玩家可在我方 9 格中自由摆放已上阵队伍。
- [ ] 游戏不包含营寨概念。
- [ ] 上排单位优先攻击敌方 `147`。
- [ ] 中排单位优先攻击敌方 `258`。
- [ ] 下排单位优先攻击敌方 `369`。
- [ ] 上排无优先目标时按 `258 -> 369` 进行次级选敌。
- [ ] 中排无优先目标时优先支援敌方单位更多的一路。
- [ ] 中排上下路敌方数量相同时，才随机采用 `147 -> 369` 或 `369 -> 147` 进行次级选敌。
- [ ] 下排无优先目标时按 `258 -> 147` 进行次级选敌。
- [ ] 不存在到达敌方营寨并造成营寨伤害的规则。
- [ ] 敌方队伍全部被击灭后，我方胜利。
- [ ] 我方队伍全部被击灭后，我方失败。
- [ ] 单位死亡后被移除。
- [ ] 波次胜利后短时状态和召唤物不会无条件带入下一波。
- [ ] 波次胜利后阵亡武将按重整规则归队，且重整代价低于正常存活。
- [ ] 同屏单位达到 60 上限时不再继续生成新单位。
- [ ] 战斗不会因卡路径、无限索敌或残留死亡单位而无法结束。
- [ ] 站位被动能按指定格位、行或列触发。
- [ ] 突进、切后和追击只改变短距离表现或目标选择，不破坏 3 路格位读性。

## 技能装配

- [ ] 每名武将拥有 6 个技能。
- [ ] 战前每名武将最多只能携带 4 个技能。
- [ ] 未携带技能不会在战斗中触发。
- [ ] 主动技能默认按冷却和目标规则自动释放。
- [ ] 玩家可在备战阶段关闭已上阵英雄主动技能自动释放。
- [ ] 关闭自动释放的可主动技能进入 HUD 技能卡牌池。
- [ ] 技能卡牌每轮最多发放 4 张，使用完当前手牌后重新抽取。
- [ ] 冷却中、系统锁定或死亡英雄所属技能不会被抽取。
- [ ] 开启自动释放的英雄技能不会进入技能卡牌池。
- [ ] 手动释放能通过时机和目标选择获得更高上限。
- [ ] 关闭更多英雄自动释放会扩大卡牌池，并降低关键技能进入当前手牌的稳定性。
- [ ] 眩晕状态下无法移动、攻击和释放技能。
- [ ] 技能目标选择符合配置。
- [ ] 至少 8 名首批武将完成 6 选 4 技能配置。
- [ ] 首批 8 名武将都有清晰的阵营、职业、定位和角色卖点。
- [ ] 首批 8 名武将每人至少有 2 套推荐技能装配。
- [ ] 第二批 8 名武将都有清晰的阵营、职业、定位和角色卖点。
- [ ] 第二批 8 名武将每人至少有 2 套推荐技能装配。
- [ ] 第三批 8 名武将都有清晰的阵营、职业、定位和角色卖点。
- [ ] 第三批 8 名武将每人至少有 2 套推荐技能装配。
- [ ] 24 名完整武将池候选都写明 2 人组合价值和 4 人组合价值。
- [ ] 16 名 MVP 武将候选都写明 2 人组合价值和 4 人组合价值。
- [ ] 每套推荐装配都能说明对应关键词和战斗风格。
- [ ] 16 名 MVP 武将候选都有表现读性、基础动作风格和关键词表现说明。
- [ ] 16 名 MVP 武将候选每人至少 2 个核心技能有明确前摇或目标提示。
- [ ] 第三批 8 名武将都有表现读性、基础动作风格和关键词表现说明。
- [ ] 第三批 8 名武将每人至少 2 个核心技能有明确前摇或目标提示。
- [ ] 突进、切后和召唤类武将不会造成 3 路站位读性混乱。
- [ ] 4 人战斗中，强技能释放前玩家能预判大致影响范围或目标。
- [ ] 装备强化表现不会遮盖角色技能本体。
- [ ] 战备能力可以根据已携带技能关键词形成有效联动。
- [ ] 默认 2 人参战条件下，至少能组成护盾反击、灼烧扩散、流血斩杀三条基础闭环。
- [ ] 4 人参战条件下，每条核心流派都有明确扩展方向。

## 关键词与装备

- [ ] 流血可以造成持续物理伤害。
- [ ] 灼烧可以造成持续法术伤害。
- [ ] 护盾可以优先吸收伤害。
- [ ] 反击可以在受击后触发额外攻击。
- [ ] 破甲可以降低防御或提高受伤。
- [ ] 连击可以造成多次命中或追加攻击。
- [ ] 士气可以提供队伍增益。
- [ ] 治疗可以恢复生命。
- [ ] 每名核心武将最多装备 2 件装备。
- [ ] 装备优先强化角色已携带技能中的关键词。
- [ ] 单件装备不会在脱离技能关键词时成为万能最优解。
- [ ] 护盾反击、灼烧扩散、流血斩杀三条流派都有装备支撑方向。
- [ ] MVP 首发装备为 12 件。
- [ ] 当前候选装备池不少于 14 件，其中 2 件可作为延后观察项。
- [ ] MVP 首发装备明确标注入选原因、覆盖流派和延后装备理由。
- [ ] MVP 首发装备经过关卡压力反查，不存在明显未覆盖压力。
- [ ] 关键高风险装备有测试关注点和微调建议。
- [ ] 每条核心流派至少有 4 件候选装备支撑。
- [ ] 每件装备都写明适配流派、适配武将、核心效果、设计目的和风险控制。
- [ ] 战备能力池覆盖关键词强化、波次针对、开战布置、职业职责、技能联动、装备联动、风险收益、资源修整 8 个维度。
- [ ] 首批战备能力池包含 30 个能力。
- [ ] 首批战备能力池普通、稀有、史诗比例约为 18/9/3。
- [ ] 战备能力不会替代技能装配、装备和羁绊决策。

## 羁绊

- [ ] 阵营羁绊按上阵武将数量激活。
- [ ] 职业羁绊按上阵武将数量激活。
- [ ] MVP 包含 4 个阵营羁绊和 4 个职业羁绊。
- [ ] 每个羁绊都定义 2 人阈值和 4 人阈值。
- [ ] 默认 2 人参战条件下，基础羁绊可以提供明确阵容方向。
- [ ] 4 人参战条件下，羁绊提供更完整的流派上限和容错，而非唯一通关门票。
- [ ] 羁绊效果能影响对应关键词或团队属性。
- [ ] 羁绊不会替代技能装配决策。
- [ ] 羁绊不会替代战备抉择和装备选择。

## 关卡、难度与数值

- [ ] 关卡结构采用“关卡 N - 难度 1-10 - 敌人波次 N”。
- [ ] 首批设计包含 4 个关卡：黄巾余烬、汜水先锋、赤壁火势、虎牢压阵。
- [ ] 每个关卡都有明确主题、主要考点、推荐构筑和解锁规则。
- [ ] 每个关卡都包含统一数量的 10 个难度。
- [ ] 同一关卡内难度按 1 到 10 依次通关解锁。
- [ ] 关卡 2 难度 1 的解锁条件为关卡 1 难度 5 通关。
- [ ] 关卡 3 难度 1 的解锁条件为关卡 2 难度 5 通关。
- [ ] 关卡 4 难度 1 的解锁条件为关卡 3 难度 7 通关。
- [ ] 每个难度都可以自由指定敌人波次数量。
- [ ] 波次是单个难度内的出怪批次，而不是固定全局数量。
- [ ] 压迫节奏不做波次重叠，仍然在当前波次结束后进入备战。
- [ ] 每个难度的波次配置需要写明敌人组合、出怪节奏和测试目的。
- [ ] 首批 4 个关卡的 10 个难度都有波次配置骨架。
- [ ] 关卡、难度、波次、敌人组有明确的策划表字段定义。
- [ ] 完成某难度全部波次才算该难度通关。
- [ ] 完成指定门槛难度后只解锁新关卡的难度 1，不自动解锁该新关卡后续难度。
- [ ] 每个关卡至少有一个主题精英或 Boss 波次。
- [ ] 敌人类型能覆盖杂兵、盾兵、弓手、破甲兵、控制兵、增益兵、精英、Boss。
- [ ] 每个首批关卡都有敌人职责表。
- [ ] 每个首批关卡都有精英或 Boss 核心机制说明。
- [ ] 每个 Boss 都有明确反制方式和流派关系。
- [ ] Boss 不完全免疫某一核心流派的全部关键词。
- [ ] 敌人职责能通过外观轮廓、站位或动作被区分。
- [ ] 破甲、灼烧、流血、护盾、眩晕等关键词有稳定的表现颜色或状态提示。
- [ ] Boss 关键技能释放前有清楚前摇、地面提示或状态变化。
- [ ] Boss 弱点窗口有明显可读表现。
- [ ] 至少 3 套构筑可以通关 MVP：灼烧爆发、护盾反击、流血斩杀。
- [ ] 单局时长目标保持在 15-25 分钟。
- [ ] 首局玩家 3 分钟内理解上阵和备战抉择。
- [ ] 首局玩家 8 分钟内完成第一次技能装配取舍。

## 商业化

- [ ] 第一版商业化只包含单局可携带参战武将数量解锁。
- [ ] 默认玩家单局最多携带 2 名参战武将。
- [ ] 付费后可额外解锁 2 个参战武将携带位。
- [ ] 付费后单局最多携带 4 名参战武将。
- [ ] 不出售武将、技能、装备、关卡通关或其他强制成长内容。
- [ ] 默认 2 人位仍可完成基础难度并体验核心玩法。
- [ ] 付费参战位不改变武将获取池或奖励稀有度。

## 局外成长

- [ ] 完成难度全部波次后展示 3 选 1 通关奖励。
- [ ] 失败时不展示通关奖励 3 选 1。
- [ ] 奖励池包含装备奖励、技能强化、战备重掷、局外资源和内容解锁。
- [ ] 武将候选通过关卡首通、门槛难度、主题试炼或局外进度解锁，不通过局内商店购买。
- [ ] 首批奖励池包含 24 个奖励条目。
- [ ] 波次后成长由战备抉择承担，难度通关后成长由奖励系统承担。
- [ ] 默认 2 人位和付费 4 人位使用同一奖励池。
- [ ] 门槛难度首通能触发新关卡解锁。
- [ ] 难度 10 首通有主题奖励或完成标记。
- [ ] 玩家通关或失败后获得局外资源。
- [ ] 首批兵书包含 12 个节点。
- [ ] 兵书节点覆盖通用稳定、战备辅助、奖励修正和阵营熟练。
- [ ] 兵书升级消耗局外资源。
- [ ] 兵书升级提供弱数值成长。
- [ ] 兵书不提供额外参战位或 4 人位专属成长。
- [ ] 局外成长不会阻断基础通关可能。
